## How does Xenora handle security?

## How are my keys and data protected?

Security is a **core design principle** at Xenora. The platform is built with a security-first approach to protect your data, users, and integrations.

### Secure by Design

* Xenora is a **SaaS-based platform**, with a **self-hosted option** available for Enterprise customers.
* Security is treated as a first-class feature across the platform.

###  Tenant Isolation & Data Protection

* Each customer (tenant) is fully **isolated** from others using strong data separation.
* Data is never shared between tenants.
* **Multi-tenant database isolation** ensures strict boundaries between customer environments.
* The platform prevents **PII / PHI data from being logged**, reducing exposure risks.

### Access Control & Authentication

* Xenora supports **Role-Based Access Control (RBAC)** to ensure only authorized users can access data.
* Each tenant manages its own users and roles.
* **Multi-Factor Authentication (MFA)** is supported for enhanced account security.
* Authentication follows modern standards:

  * **JWT-based authentication**
  * Compatible with **OAuth 2.0 and OpenID Connect**
  * **API Token authentication** for secure system integrations

###  Key & Sensitive Data Security

* Sensitive information, including **Bring Your Own Keys (BYOK)**, is securely encrypted.
* Keys and confidential data are stored and handled using industry-standard security practices.

### Compliance

* **Primary Compliance Standard:** **HIPAA**
* Designed to support regulated workloads requiring strong privacy and data protection controls.

###  Enterprise Deployment Options

* Xenora can be deployed within your own infrastructure for organizations with advanced security or compliance requirements (Enterprise plans only).
