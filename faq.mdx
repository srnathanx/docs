## How Xenora works

<AccordionGroup>
  <Accordion title="How does Xenora work technically?" icon="microchip">
    Xenora is an **agentic Retrieval-Augmented Generation (RAG) platform** designed to understand, organize, and reason over your knowledge.

    **Knowledge ingestion** — Xenora builds a knowledge base from documents, URLs, PDFs, Excel and CSV files, Markdown files, and cloud storage.

    **Knowledge processing & storage** — ingested content is processed, structured, and converted into embeddings, then stored in a **vector database** for fast and accurate semantic retrieval.

    **Agentic retrieval & query understanding** — when a user submits a query, Xenora uses agentic retrieval to understand intent and pull relevant information from the knowledge base.

    **Response generation** — the retrieved knowledge is combined with the user's query and context. Xenora then generates a clear, accurate, and context-aware response using large language models.
  </Accordion>

  <Accordion title="How does Xenora work in simple terms?" icon="lightbulb">
    Xenora helps you turn your existing information into a smart, AI-powered assistant that can answer questions accurately and instantly.

    1. **Add your knowledge** — documents, PDFs, website URLs, spreadsheets, and Markdown notes. Xenora securely organizes them into a single knowledge base.
    2. **Xenora learns your content** — it reads, understands, and structures your information so it can be searched efficiently — without changing your originals.
    3. **Ask questions in natural language** — no special commands or technical knowledge required.
    4. **Find the right information** — Xenora intelligently searches your knowledge base for the most relevant content.
    5. **Get clear answers** — combining your question with the retrieved content, Xenora delivers a clear, context-aware response right when you need it.
  </Accordion>
</AccordionGroup>

## Security & compliance

<AccordionGroup>
  <Accordion title="How does Xenora handle security?" icon="shield-halved">
    Security is a **core design principle** at Xenora. The platform is built with a security-first approach to protect your data, users, and integrations.

    **Secure by design** — Xenora is a SaaS-based platform, with a self-hosted option available for Enterprise customers. Security is treated as a first-class feature across the platform.

    **Tenant isolation & data protection** — each customer (tenant) is fully isolated using strong data separation. Data is never shared between tenants, multi-tenant database isolation enforces strict boundaries, and the platform prevents PII / PHI data from being logged.

    **Access control & authentication** — Xenora supports Role-Based Access Control (RBAC), per-tenant user and role management, and Multi-Factor Authentication (MFA). Authentication uses **JWT-based authentication**, **OAuth 2.0**, **OpenID Connect**, and **API token authentication** for secure system integrations.

    **Key & sensitive data security** — sensitive information, including **Bring Your Own Keys (BYOK)**, is securely encrypted using industry-standard practices.
  </Accordion>

  <Accordion title="What compliance standards does Xenora meet?" icon="circle-check">
    **Primary compliance standard:** **HIPAA**. Xenora is designed to support regulated workloads requiring strong privacy and data protection controls. For organizations with advanced security or compliance requirements, Xenora can be deployed within your own infrastructure (Enterprise plans only).
  </Accordion>
</AccordionGroup>
